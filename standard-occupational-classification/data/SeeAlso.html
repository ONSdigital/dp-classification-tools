SOC “see also” entries
When 'see', 'see also' and 'see notes' is given instead of a code for a job title, it is an indication that more action is required to find a suitable code. There are several reasons for this:
  • the job title has alternative spellings, for example 'Advisor' see 'Adviser';
  • the job title searched for is similar to other terms for the job, such as 'Minder, machine' see 'Machinist', so users should search for 'Machinist';
  • more information is required to code. If a response is returned with '( )' it means there are additional terms required to provide a code, for example if you search for 'Manager', it will return 'Manager ( )'. In this instance the coding tool would require to know what it is that's being managed, or the industry, so for example 'Manager (catering)'.

Where 'see also' is returned it means that an alternative indexing word might be appropriate, so for example,
  • 1251, “Factor, estate (Scotland)”
  • 3234, “Factor, housing (Scotland: local government)”
  • 1251, “Factor, housing (Scotland)”
  • 1254, “Factor, motor”
  • “Factor” see also Dealer

In this example, one would use the Dealer list after checking the entries for Factor.

When 'See notes' is presented, it requires users to refer to the SOC 2010 coding index, and to read the relevant notes for further information.

Should users require assistance with coding for these job titles, please refer to the Standard Occupational Classification (SOC) 2010 Volume 2 coding index
